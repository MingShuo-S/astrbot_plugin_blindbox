## v0.7.0 (2024-05-12)

### 新功能

#### 每周一次盲盒任务选择机制

本版本对盲盒抽取流程进行了重大更新，实现了以下功能：

1. **三选一流程**
   - 抽盲盒时不再直接抽取一个任务，而是显示 3 个任务选项
   - 用户通过回复 `1`、`2` 或 `3` 来选择心仪的任务
   - 选择超时 5 分钟后自动过期，需要重新抽取

2. **每周一次限制**
   - 每个小组每周只能抽取一次盲盒
   - 除非满足以下条件之一，否则无法抽取新任务：
     - 上周任务已完成（已通过审核）
     - 上周任务已拒绝（已审核）
     - 距离上周抽取已超过 7 天

3. **任务有效期**
   - 每个选定的任务从选择时刻起有效期为 **7 天**
   - 7 天内需要提交任务成果
   - 超过 7 天未提交也允许重新抽取

4. **提交后重置**
   - 任务通过审核后，可以抽取下一周的任务
   - 任务被拒绝后，可以重新抽取或继续完成该任务

### 技术变更

- 新增 `_pick_three_tasks()` 方法，用于随机选择 3 个不同的任务
- 新增 `_confirm_selection()` 方法，用于处理用户的选择并更新任务状态
- 新增 `_can_draw_again()` 方法，用于检查是否可以抽取新任务
- 新增 `_handle_selection_response()` 方法，用于处理用户的数字回复
- 新增 `handle_selection_choice()` 事件处理器，用于捕获数字消息
- 修改 `_draw_for_group()` 方法的返回值和逻辑
- 更新状态结构：添加 `pending_selections` 字段用于存储待选择的任务选项
- 添加 `_user_selections` 属性用于跟踪用户的选择状态
- 修改 `_normalize_state()` 方法以处理过期的待选择记录

### API 变更

- `GET /astrbot_plugin_blindbox/group/redraw` 现在返回三个任务选项和 `selection_id`，而不是直接返回确定的任务

### 帮助文本更新

- 更新了 `/blindbox` 指令说明，包含选择流程说明
- 补充说明了每周限制和任务有效期

### 文档更新
